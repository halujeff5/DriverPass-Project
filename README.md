# DriverPass Project 🚗 🚦 

### DriverPass is a project done for a stakeholder who wanted to make passing the road test for new drivers easier. It contains a step by step guide for it to help driver succeed. It is to be accessible online allowing multiple users simultaneously along with an interface for admins and instructors of the driving school. 🚥

### The UML diagrams created are comprehensive and fully capture the necessary components for development and allows the client to see their envisioned features come to play.The UML diagrams contain class diagrams, sequential diagrams, and case diagrams to illustrate the functions of the project as demanded by client.

### If I could redo any portion of the project however, I would revise the case diagram to fully reflect the users of the system, which includes the stakeholders, admins, instructors, and students. 

### To interpret the user's needs we conducted a informational interview with the client where careful notes were taken. The whole purpose of the interview was to get specific gudielines for the project and make sure they align with business goals. The client must be able to see their "vision" come to life with a CI/CD pipeline that has been documented via a Gantt chart.

### Software design can be approached first with UML diagrams, a schedule can be created via a Gantt chart. UML diagrams can describe the use cases, sequential flow, and even database design with the Class UML. The interview will be a crucial part of the system design where I gather client requirements, goals, and feedback.
